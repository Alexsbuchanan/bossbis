# BossBiS v0.1.5 — NPC-vs-Player (damage taken) Port + Parity Plan

> Conventions identical to v0.1.1–v0.1.4 (PORT-style, `strictfp`, no RNG, `long` in roll math,
> `Math.trunc`→`(int)`/`(long)` cast, no `new Gson()` in main, per-task commits
> `port: … <weirdgloop@5e313c1>`, no push, Gradle `--no-daemon --max-workers=1` on WSL2). Upstream at
> `tools/upstream/src/`. Branch: `feat/v0.1.5-nvp` (already checked out — do NOT switch).

**Goal:** Port the REVERSE direction — `NPCVsPlayerCalc` (`tools/upstream/src/lib/NPCVsPlayerCalc.ts`,
274 lines) → `src/main/java/org/bossbis/calc/NpcVsPlayerCalc.java` extending `BaseCalc` — i.e. how much
damage the NPC does to the player / damage taken. Then parity-verify it the same way the player-vs-npc
calc was verified: a `kind:"nvp"` sweep over (monster attacker × player defender) scenarios, run through
the REAL upstream `calculateNpcVsPlayer` (already present in upstream `TestUtils.ts`), captured as
`expected` and asserted by `ParityCorpusTest` at integer-exact / float-1e-9 tolerance. The existing 318
player-vs-npc rows must keep passing.

**Tolerance:** integer outputs (`npcMaxHit`, `npcMaxAttackRoll`, `playerDefRoll`) EXACT; floating outputs
(`npcAccuracy`, `npcDps`, `avgDmgTaken`) relative ≤ 1e-9 (the runner already does relative for floats).

## Milestone 1 — `NpcVsPlayerCalc` port + constants

**Files:** new `src/main/java/org/bossbis/calc/NpcVsPlayerCalc.java`; modify `Constants.java`
(add `ALWAYS_ACCURATE_MONSTERS`, `NPC_HARDCODED_MAX_HIT`); new test `NvpTest.java` + a fixtures helper.

**Source:** `NPCVsPlayerCalc.ts` end-to-end. Port faithfully:
- `getPlayerDefensiveBonus()` (:45-67) — switch on the (normalized lowercase) `monster.style` into the
  player's `defensive.{crush,stab,slash,magic,ranged}`.
- `getPlayerDefenceRoll()` (:127-167) — stance bonus (Defensive/Controlled/Longrange), effective level =
  `def + boost.def`, the **factorDefence prayer numerator sum** (`Σ (factorDefence[0]-100)` then
  `trackFactor([numerator+100, 100])`), Torag's missing-health factor, the magic-style branch
  (`effectiveMagicLevel*7/10 + effectiveLevel*3/10`, with `factorDefenceMagic` prayers where
  `combatStyle==='magic'`), `+ stanceBonus`, gear bonus `= bonus + 64`, final `(8 + effectiveLevel) * gearBonus`.
- `getNPCMaxAttackRoll()` (:172-194) — per-style `(9 + skill) * (offBonus + 64)`.
- `getNPCMaxHit()` (:199-236) — per-style `trunc(((9+skill)*(strBonus+64)+320)/640)`, the
  `NPC_HARDCODED_MAX_HIT` override, the magic wiki-maxHit override, and the per-monster reduced-max-hit
  cases (Aberrant spectre/Dust devil/Banshee/Wall beast/Sourhog/troll-Neitiznot).
- `getHitChance()` (:241-251) — `ALWAYS_ACCURATE_MONSTERS` → 1; else
  `BaseCalc.getNormalAccuracyRoll(getNPCMaxAttackRoll(), getPlayerDefenceRoll())`.
- `getDistributionImpl()`/`getDistribution()` (:26-115) — `HitDistribution.linear(acc, 0, max)` wrapped in
  an `AttackDistribution`, then the NPC→player transforms: **Elysian spirit shield** (30% no reduction +
  70% reduced by `max(1, trunc(d/4))`), **Dinh's bulwark Block** (×0.8), **Justiciar armour**
  (`trunc(damage * (1 - bonus/3000))`). Memoize. The commented-out overhead-prayer block stays dropped
  (the prayer-through monsters are deferred upstream too).
- `getDpt()` (:256-258) — `getDistribution().getExpectedDamage() / monster.speed`.
- `getDps()` (:263-265) — `getDpt() / SECONDS_PER_TICK`.
- `getPlayerVsNPCCalc()` (:34-43) — a memoized `PlayerVsNpcCalc(player, monster, {loadoutName:…/forward,
  disableMonsterScaling:true})`; needs the `EquipmentRepository`/`SpellRepository` linkage (the Java
  `PlayerVsNpcCalc` takes injected repos — thread them through the `NpcVsPlayerCalc` constructor).
- `getAverageDamageTaken()` (:270-273) — `ttk = getPlayerVsNPCCalc().getTtk(); return ttk ? ttk*getDps()
  : undefined`. Model JS `ttk ? …` truthiness: return a `double`; if ttk is `0`/NaN/Infinity treat as the
  JS falsy case → return `Double.NaN` (the sweep already drops non-finite expected fields, so an
  `avgDmgTaken` only appears in the corpus when ttk was finite & nonzero).

**Monster style normalization (critical):** upstream `monster.style` is a single lowercase
`CombatStyleType` derived in `getMonsters()` as `m.style.join(',').toLowerCase()` kept iff it is one of
`stab|slash|crush|magic|ranged` (else null). The Java `Monster.getStyle()` returns the RAW capitalized
`List<String>` (e.g. `["Crush"]`). Add a private `monsterStyle()` helper to `NpcVsPlayerCalc` that
replicates `join(',').toLowerCase()` + the `isCombatStyleType` validity gate, returning null otherwise.
All `monster.style` reads go through it.

**Leagues:** none in NPCVsPlayerCalc.ts (no leagues branches to drop). The class-level deviation note
records that the `player.leagues` paths inherited via `BaseCalc` are already dropped (spec §4.2).

- [ ] TDD: `NvpTest` — build an NPC-attacker-vs-player calc for a known monster (e.g. Abyssal demon

  upstream `calculateNpcVsPlayer`. Commit `port: NpcVsPlayerCalc (damage taken) <weirdgloop@5e313c1>`.

## Milestone 2 — NvP parity sweep generator

**Files:** new `tools/gen-nvp-sweep.ts`; modify `tools/package.json` (add `gen-nvp-sweep` script);
modify `tools/gen-reference-corpus.ts` (accept the `kind:"nvp"` discriminator + nvp expected fields).

**Approach:** mirror `gen-parity-sweep.ts` but call upstream `calculateNpcVsPlayer(monster, player)` from
`@/tests/utils/TestUtils` (already exports `npcMaxAttackRoll`, `npcMaxHit`, `npcDps`, `npcAccuracy`,
`playerDefRoll`). For `avgDmgTaken`, construct `new NPCVsPlayerCalc(player, monster, {loadoutName:'test'})`
and call `getAverageDamageTaken()`. Build ~30–80 rows = several monster attackers (melee/ranged/magic,
varied max hit + attack style) × several player defensive setups (tank armour, Protect-from-X prayer
[note: not yet modeled by upstream NvP — prayers don't change NvP output here, so vary defence instead],
varied defence). Tag `kind:"nvp"`, `exercises:["NpcVsPlayerCalc.dps"]`, capture
`{playerDefRoll, npcMaxHit, npcMaxAttackRoll, npcAccuracy, npcDps, avgDmgTaken}` as `expected`.
The player's WEAPON/style still matters because `getPlayerDefenceRoll` reads the player's stance and the
monster's attack style picks the defensive bonus; pick defensive-leaning gear sets (tank armour) to
exercise `flat_armour`-free defence + Justiciar/Elysian/Dinh's transform branches.

**Self-check:** before writing rows, reproduce a KNOWN nvp value (probe one upstream
`calculateNpcVsPlayer` result by hand first, hard-code it in the generator's `selfCheck()`), failing
loudly if the loader chain / pin drifts.

**`gen-reference-corpus.ts` changes:** allow an optional `kind` field (`"pvn"` default | `"nvp"`); when
`kind==="nvp"`, validate the nvp expected keys instead of pvn keys. Keep the finite-number requirement.
