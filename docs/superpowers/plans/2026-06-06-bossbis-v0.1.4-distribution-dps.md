# BossBiS v0.1.4 — Distribution + DPS/TTK Implementation Plan

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Conventions identical to v0.1.1–v0.1.3 (PORT-style, `strictfp`, no RNG, `long` in roll math, `Math.trunc`→`(int)`/`(long)`, per-task TDD + `port: … <weirdgloop@5e313c1>` commits, no push, Gradle `--no-daemon --max-workers=1` on WSL2). Upstream at `tools/upstream/src/`.

**Goal:** Port the hit-distribution pipeline — `HitDist` (AttackDistribution/HitDistribution/WeightedHit/Hitsplat + transformers + `getHash`), the `dists/` (claws, bolts), `getAttackerDist` + `applyNpcTransforms` + `getDistribution`, `getMax`/`getExpectedDamage`, and `getDpt`/`getDps`/`getHtk`/`getTtk`. This makes `getMax()` (the TRUE distribution top, which is upstream's `maxHit` source) and `getDps`/`getTtk` available, **lights up blisterwood** (its vampyrebane ×5/4 is an NPC transform) and the dps/ttk corpus rows, and switches the corpus `maxHit` assertion from `getMaxHit()` (base) to `getMax()` (distribution).

**Branch:** `feat/v0.1.4-distribution` off `main` @ `v0.1.3`. **Distribution-tolerance:** integer outputs exact; probabilities/dps/ttk relative ≤1e-9 (the runner already does relative for floats). Determinism rules (spec §4.7) are load-bearing here: `getHash` via `BigInteger`; the delay pack/unpack signed-`int` with the verbatim `0x8F000000` mask; `support/D3.sum` left-to-right order; integer powers as repeated multiplication; TTK fold epsilon `TTK_DIST_EPSILON=1e-4`, cap `TTK_DIST_MAX_ITER_ROUNDS=1000` (in `Constants`).

**Definition of done:** `./gradlew build` green; **every existing `maxHit` row passes via `getMax()`** (melee/ranged/magic), **blisterwood now passes** (55, via the distribution transform), new **dps + ttk** corpus rows pass (≤1e-9), all accuracy/attack/defence rows still pass, `PortStatusTest` green; determinism OK; jar <10 MiB; `PlayerVsNpcCalc.{distribution,dps,ttk}` + `HitDist` + `dists` → `ported`; CI green on `main`; tag `v0.1.4`.
