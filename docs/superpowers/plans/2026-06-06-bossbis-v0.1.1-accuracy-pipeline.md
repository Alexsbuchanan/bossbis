## How to read this plan (PORT — same conventions as the v0.1.0 plan)

Source of truth = weirdgloop's TS at `tools/upstream/src/`. Each task gives: target Java file(s), the **upstream source `file:line` to port from**, the contract/structure, the **gate** (a parity-corpus row going green and/or a targeted unit test with full code), and the **gotchas**. For logic-heavy methods (the attack rolls), the plan cites the exact source method and the ordered branch list rather than reproducing ~130 lines of TS — the implementer transcribes faithfully and the corpus row is the objective gate. **Global rules (carried from v0.1.0, still mandatory):**
- `strictfp` on every calc class doing floating-point. **No RNG** (CI guard). Integer rolls use **`long` intermediates** (`trackFactor` multiplies before dividing and can exceed 2³¹). `Math.trunc` → `(long)`/`(int)` truncation, NOT `Math.floor` (except where upstream floors).
- Faithful first, optimize never (this phase). Match upstream method/branch structure 1:1 for reviewability.
- Per task: write/extend test → run (fail) → port → run (pass) → commit (`port: <thing> <weirdgloop@5e313c1>`), then flip the relevant `PORT_STATE.md` + `port-state.json` entries when a path's corpus rows pass. Do NOT push (controller manages branch/PR).

**Parity tolerance (§5.3):** integer outputs (`maxAttackRoll`, `npcDefRoll`) **exact**; floating outputs (`accuracy`/`hitChance`) **relative ≤ 1e-9**.

```java
// Port of BaseCalc.getNormalAccuracyRoll (BaseCalc.ts:155-168). Pure double math; strictfp.
public static strictfp double getNormalAccuracyRoll(int atkIn, int defIn)
{
	double atk = atkIn, def = defIn;
	if (atk < 0) atk = Math.min(0, atk + 2);
	if (def < 0) def = Math.min(0, def + 2);
	if (atk >= 0 && def >= 0) return stdRoll(atk, def);
	if (atk >= 0 && def < 0)  return 1 - 1 / (-def + 1) / (atk + 1);
	if (atk < 0 && def >= 0)  return 0;
	if (atk < 0 && def < 0)   return stdRoll(-def, -atk);
	return 0;
}
private static strictfp double stdRoll(double attack, double defence)
{
	return attack > defence
		? 1 - ((defence + 2) / (2 * (attack + 1)))
		: attack / (2 * (defence + 1));
}
```
